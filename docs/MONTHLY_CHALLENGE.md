# Monthly Challenge

TwinBench should become a public benchmark people want to compete on.

The monthly challenge format is the simplest recurring loop:

- publish the current reference board
- invite new submissions for the month
- highlight the strongest new result
- publish one short summary of what changed

Recommended public labels:

- `Top Score This Month`
- `Most Improved`
- `Reference Runtime`
- `Verified Artifact`

Do not start with closed certification language. The point is participation and trust, not gatekeeping.

## Suggested Cadence

Week 1:
- announce the challenge
- highlight the current reference result

Week 2–3:
- collect submissions
- answer compatibility questions

Week 4:
- publish results recap
- spotlight one runtime author or improvement story

## Suggested Prize Shapes

- homepage spotlight
- case study feature
- benchmark interview
- launch thread mention

Keep the reward social and reputational before making it financial.

# TwinBench Roadmap

TwinBench should grow in the order that best builds trust and participation.

## Now

- public repo
- public benchmark docs
- reference runtime artifact
- static website and leaderboard surface
- one-click demo path
- outreach wave materials

## Next

- hosted-assistant support with explicit result classes
- first non-Nullalis public result
- GitHub Discussions-based community loop
- monthly benchmark challenge

## Later

- framework wrappers
- richer comparison pages
- benchmark SDKs
- broader major-lab participation

## Guiding Rule

TwinBench should always become easier to run without becoming easier to game.

# Outreach Waves

TwinBench should be introduced in deliberate waves, not all at once.

## Wave 1: Direct Competitors

Targets:

- OpenClaw
- NanoBot
- PicoClaw
- Letta

Goal:

- get the first serious external runtime teams to react
- invite artifact-backed competition
- normalize TwinBench as a benchmark people publicly answer

Message emphasis:

- open challenge
- no editorial scoring
- compatibility notes are handled honestly
- publish your own artifact

Template:

- [EMAIL_CAMPAIGN.md](EMAIL_CAMPAIGN.md) -> `Wave 1: Direct Competitors`

## Wave 2: Adjacent Platforms

Targets:

- Mem0
- LangGraph / LangChain runtime teams
- other long-lived agent infrastructure teams

Goal:

- widen legitimacy beyond direct competitors
- get critique on methodology and integration paths
- attract framework and infra teams who shape the ecosystem

Message emphasis:

- category definition
- methodology and evidence separation
- partial compatibility is still useful

Template:

- [EMAIL_CAMPAIGN.md](EMAIL_CAMPAIGN.md) -> `Wave 2: Adjacent Platforms / Infrastructure`

## Wave 3: Frontier Labs

Targets:

- OpenAI
- Anthropic
- Google DeepMind
- xAI

Goal:

- position TwinBench as the runtime benchmark category labs should know exists
- invite critique, participation, or partial reference runs
- create reputational gravity around the category

Message emphasis:

- runtime evaluation gap
- openness and reproducibility
- personal AI assistant runtime category

- [EMAIL_CAMPAIGN.md](EMAIL_CAMPAIGN.md) -> `Wave 3: Major Labs`

## Send Criteria

Send only after:

- the README front door is stable
- the current reference result is clear
- the progression page is clean
- the agent-run guide is usable
- the GitHub metadata is updated

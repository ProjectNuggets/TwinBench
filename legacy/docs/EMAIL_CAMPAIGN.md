# Email Campaign

Use these templates when inviting runtime teams, adjacent platforms, and major AI labs to run against TwinBench.

## Core Ask

Keep this sentence stable across every wave:

TwinBench is open, evidence-first, vendor-neutral, and built for personal AI assistant runtimes. We invite you to run your system and publish an artifact-backed result.

## Wave 1: Direct Competitors

Subject: Invitation to run your runtime against TwinBench

Hi,

We think a new benchmark category is needed for personal AI assistant runtimes.

TwinBench is an open, evidence-first, vendor-neutral benchmark for runtimes that aim to remember, act, follow up, stay safe, and operate over time. We are inviting serious runtime builders to run their systems and publish artifact-backed results.

If your runtime already exposes the benchmark contract, we would love to include a verified submission from your team. If not, we are still happy to work from a documented compatibility path. The benchmark now distinguishes missing bootstrap, contract mismatch, and unsupported surfaces instead of flattening them into a bad score.

Repo: https://github.com/ProjectNuggets/DTaaS-benchmark

Best,

## Wave 2: Adjacent Platforms / Infrastructure

Subject: Invitation to evaluate your runtime platform with TwinBench

Hi,

We think a new benchmark category is needed for personal AI assistant runtimes, and your platform sits close enough to the runtime layer that your feedback would be useful even if full native compatibility is not there yet.

TwinBench is open, evidence-first, and vendor-neutral. It measures memory, autonomous execution, cross-channel behavior, safety, resilience, and artifact-backed runtime behavior over time.

If your platform is only partially compatible today, a documented compatibility path or partial artifact is still valuable.

Repo: https://github.com/ProjectNuggets/DTaaS-benchmark

Best,

## Wave 3: Major Labs

Subject: TwinBench: open benchmark for personal AI assistant runtimes

Hi,

We believe the benchmark landscape still under-measures the runtime behavior required for persistent personal AI assistants. TwinBench is an attempt to define that category openly and rigorously.

We would be glad to see your team run a compatible runtime, research system, or hosted assistant product against the benchmark and publish an artifact-backed result. Even if your current product does not match the full native contract, your team’s feedback on the category and evaluation method would be valuable.

Repo: https://github.com/ProjectNuggets/DTaaS-benchmark

Best,

Subject: Request for feedback on TwinBench

Hi,

We are publishing TwinBench as an open benchmark for personal AI assistant runtimes. The benchmark focuses on memory, autonomous execution, cross-channel behavior, safety, scale, resilience, and artifact-backed evaluation.

We would value critique, feedback, and independent runs.

Repo: https://github.com/ProjectNuggets/DTaaS-benchmark

## Follow-Up Email

Subject: Following up on TwinBench invitation

Hi,

Following up on TwinBench in case it is relevant to your team. If your runtime is not directly contract-compatible yet, the repo now includes compatibility, preflight, and integration-path docs to make adoption easier.

If useful, we are happy to review a compatibility path before you spend time on a full run.
